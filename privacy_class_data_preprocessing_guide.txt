개인정보 제공 태도 분석 데이터 처리 이용 가이드

파일 목적
- 이 문서는 privacy_class_regression_statsmodels.ipynb에서 사용한 핵심 변수의 원척도, 전처리 방식, 역코딩 방향, 분석 시 해석 기준을 정리한 것이다.
- 분석에서는 해석 편의를 위해 일부 변수를 역코딩하였다. 따라서 회귀분석과 로지스틱 분석 결과를 해석할 때 원변수와 분석변수의 방향을 구분해야 한다.


1. 종속변수: 개인정보 제공 태도

1) GIVEPIFR
- 원변수 의미: 할인이나 무료상품을 받기 위해 개인정보를 제공할 것인지에 대한 태도
- 원척도:
  1 = 매우 동의
  2 = 동의
  3 = 동의도 반대도 아님
  4 = 반대
  5 = 매우 반대
- 원척도 해석:
  값이 낮을수록 개인정보 제공에 동의한다.
  값이 높을수록 개인정보 제공에 반대한다.
- 분석 처리:
  GIVEPIFR_R = 6 - GIVEPIFR
- 분석변수 해석:
  GIVEPIFR_R 값이 높을수록 개인정보 제공 동의 수준이 높다.
  1 = 매우 반대
  5 = 매우 동의
- 회귀분석 사용 변수:
  GIVEPIFR_R
- 로지스틱 분석 사용 변수:
  GIVEPIFR_agree
  원척도 GIVEPIFR가 1 또는 2이면 1, 원척도 GIVEPIFR가 3, 4, 5이면 0으로 코딩한다.
  따라서 GIVEPIFR_agree = 1은 개인정보 제공에 동의한다는 의미이다.

2) GIVEPIPC
- 원변수 의미: 민간기업이 나의 개인정보를 사용하여 이익을 내더라도 개인정보를 제공할 것인지에 대한 태도
- 원척도:
  1 = 매우 동의
  2 = 동의
  3 = 동의도 반대도 아님
  4 = 반대
  5 = 매우 반대
- 원척도 해석:
  값이 낮을수록 개인정보 제공에 동의한다.
  값이 높을수록 개인정보 제공에 반대한다.
- 분석 처리:
  GIVEPIPC_R = 6 - GIVEPIPC
- 분석변수 해석:
  GIVEPIPC_R 값이 높을수록 개인정보 제공 동의 수준이 높다.
  1 = 매우 반대
  5 = 매우 동의
- 회귀분석 사용 변수:
  GIVEPIPC_R
- 로지스틱 분석 사용 변수:
  GIVEPIPC_agree
  원척도 GIVEPIPC가 1 또는 2이면 1, 원척도 GIVEPIPC가 3, 4, 5이면 0으로 코딩한다.
  따라서 GIVEPIPC_agree = 1은 개인정보 제공에 동의한다는 의미이다.


2. 핵심 독립변수: 경제적 자원 및 계층 변수

1) INCOM0
- 원변수 의미: 월평균 가구소득 원자료/연속형 변수
- 분석 처리:
  log_INCOM0 = log(INCOM0 + 1)
- 분석변수 해석:
  log_INCOM0 값이 높을수록 월평균 가구소득이 높다.
- 주 분석 사용 방식:
  주 분석의 객관적 계층 변수로 log_INCOM0를 사용한다.
- 회귀분석 해석:
  log_INCOM0의 계수가 양수이면 소득이 높을수록 개인정보 제공 동의 수준이 높다는 방향이다.
  log_INCOM0의 계수가 음수이면 소득이 높을수록 개인정보 제공 동의 수준이 낮다는 방향이다.

2) INCOME
- 원변수 의미: 월평균 가구소득 범주형 변수
- 분석 처리:
  INCOME_CAT으로 범주형 처리하여 강건성 분석에서 사용한다.
- 분석변수 해석:
  소득 측정 방식을 연속형 로그소득이 아니라 범주형 소득으로 바꾸어도 결과가 유지되는지 확인하는 용도이다.
- 사용 위치:
  주 분석이 아니라 연구 마지막의 강건성 분석에서 사용한다.

3) RANK
- 원변수 의미: 주관적인 계층의식
- 원척도:
  1 = 최하층
  10 = 최상층
- 분석 처리:
  별도 역코딩하지 않는다.
- 분석변수 해석:
  값이 높을수록 주관적 계층의식이 높다.
- 회귀분석 해석:
  RANK의 계수가 양수이면 주관적 계층의식이 높을수록 개인정보 제공 동의 수준이 높다는 방향이다.
  RANK의 계수가 음수이면 주관적 계층의식이 높을수록 개인정보 제공 동의 수준이 낮다는 방향이다.

4) SATFIN
- 원변수 의미: 가계상태 만족도
- 원척도:
  1 = 매우 만족
  2 = 만족
  3 = 보통
  4 = 불만족
  5 = 매우 불만족
- 원척도 해석:
  값이 낮을수록 가계상태 만족도가 높다.
  값이 높을수록 가계상태 만족도가 낮다.
- 분석 처리:

- 분석변수 해석:

  1 = 매우 불만족
  5 = 매우 만족
- 회귀분석 및 로지스틱 분석 사용 변수:
  SATFIN_R
- 회귀분석 해석:
  SATFIN_R의 계수가 양수이면 가계상태 만족도가 높을수록 개인정보 제공 동의 수준이 높다는 방향이다.
  SATFIN_R의 계수가 음수이면 가계상태 만족도가 높을수록 개인정보 제공 동의 수준이 낮다는 방향이다.

5) FINPROS
- 원변수 의미: 10년 이내 가계경제상태 전망
- 원척도:
  1 = 상당히 좋아질 것이다

- 분석변수 해석:

  1 = 상당히 나빠질 것이다

3. 통제변수

1) AGE
- 원변수 의미: 응답자 연령
- 분석 처리:
  별도 역코딩하지 않는다.
- 분석변수 해석:
  값이 높을수록 연령이 높다.

2) SEX
- 원변수 의미: 응답자 성별
- 원척도:
  1 = 남자
  2 = 여자
- 분석 처리:
  회귀식에서 C(SEX)로 범주형 처리한다.
- 분석변수 해석:
  statsmodels 회귀표에서 C(SEX)[T.2.0]은 기준범주인 남자와 비교한 여자의 차이를 의미한다.

3) EDUC
- 원변수 의미: 응답자 최종 학력
- 원척도:
  0 = 무학
  1 = 초등학교
  2 = 중학교
  3 = 고등학교
  4 = 전문대학
  5 = 대학교
  6 = 석사
  7 = 박사
  8 = 서당한학
- 분석 처리:
  현재 노트북에서는 연속형/순서형 통제변수처럼 투입한다.
- 주의:
  EDUC=8인 서당한학은 일반적인 학력 순서와 완전히 일치하지 않을 수 있으므로 해석 시 주의가 필요하다.

4) ABLITINF

  1 = 매우 높음
  2 = 높음
  3 = 보통
  4 = 낮음
  5 = 매우 낮음
- 원척도 해석:
  값이 낮을수록 정보판별능력이 높다.
  값이 높을수록 정보판별능력이 낮다.
- 결측 처리:
  ABLITINF 원변수의 -1은 비해당 값으로 보고 결측 처리한다.
- 분석 처리:
  ABLITINF_R = 6 - ABLITINF
- 분석변수 해석:
  ABLITINF_R 값이 높을수록 정보판별능력이 높다.
  1 = 매우 낮음
  5 = 매우 높음
- 회귀분석 및 로지스틱 분석 사용 변수:
  ABLITINF_R
- 회귀분석 해석:
  ABLITINF_R의 계수가 양수이면 정보판별능력이 높을수록 개인정보 제공 동의 수준이 높다는 방향이다.
  ABLITINF_R의 계수가 음수이면 정보판별능력이 높을수록 개인정보 제공 동의 수준이 낮다는 방향이다.

5) TECHHARM

- 분석변수 해석:

5. 결과 해석의 기본 원칙

- 회귀분석에서 종속변수 GIVEPIFR_R, GIVEPIPC_R는 값이 높을수록 개인정보 제공 동의 수준이 높다는 뜻이다.
- 따라서 독립변수의 계수가 양수이면 해당 변수가 증가할수록 개인정보 제공 동의 수준이 높아지는 방향이다.
- 독립변수의 계수가 음수이면 해당 변수가 증가할수록 개인정보 제공 동의 수준이 낮아지는 방향이다.
- 로지스틱 분석에서 종속변수 GIVEPIFR_agree, GIVEPIPC_agree는 개인정보 제공 동의 여부이다.
- 로지스틱 분석에서 계수가 양수이면 해당 변수가 증가할수록 개인정보 제공에 동의할 가능성이 높아지는 방향이다.
- 로지스틱 분석에서 계수가 음수이면 해당 변수가 증가할수록 개인정보 제공에 동의할 가능성이 낮아지는 방향이다.
- SATFIN_R, FINPROS_R, ABLITINF_R는 모두 해석 방향을 긍정적으로 맞춘 변수이다.
- TECHHARM_R는 긍정 방향이 아니라 기술위험 인식이 높아지는 방향으로 맞춘 변수이다.
- INCOME은 주 분석 변수가 아니라 강건성 분석에서만 사용한다.
